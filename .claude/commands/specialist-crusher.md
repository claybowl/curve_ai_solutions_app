# Crusher - System Health & Monitoring Command

Engage Crusher for system health monitoring, performance diagnostics, incident response, and reliability engineering.

## Usage

```
/crusher [health check | performance diagnostic | monitoring setup | incident response]
```

## Examples

**System Health Check:**
```
/crusher Full system health diagnostic. Check database performance, API response times, and error rates. Report any anomalies.
```

**Monitoring Setup:**
```
/crusher Design comprehensive monitoring system for AI agent execution. Include alerting thresholds and dashboard requirements.
```

**Performance Diagnostics:**
```
/crusher Investigate slow query performance during peak hours. Identify root cause and optimization opportunities.
```

**Incident Response:**
```
/crusher Red Alert: Database connection pool exhaustion detected. Implement emergency scaling and provide post-mortem analysis.
```

## Capabilities

- System health monitoring and diagnostics
- Performance profiling and optimization
- Incident response and post-mortem analysis
- Reliability engineering and SRE practices
- Monitoring system design and implementation
- Alerting strategy and runbook creation
- Capacity planning and scaling analysis
- Error tracking and debugging

## Coordination

Crusher provides real-time system insights with actionable recommendations. For infrastructure changes, Crusher coordinates with Geordi through #1.
