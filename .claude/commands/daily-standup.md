# Daily Standup Command

Conduct daily 15-minute standup meeting to coordinate priorities and synchronize the team.

## Usage

```
/daily-standup [optional: specific focus areas or blockers]
```

## Examples

**Standard Daily Standup:**
```
/daily-standup
```

**Focused Standup:**
```
/daily-standup Focus on AI agent platform launch preparation. Identify any blockers.
```

**Blocker Resolution:**
```
/daily-standup Critical blocker: Database migration delayed. Need immediate coordination.
```

## Standup Structure

**Order:**
1. **Captain** (30 seconds): Strategic priorities for the day
2. **#1** (1 minute): Critical path items and coordination needs
3. **Specialists** (1 minute each, in priority order):
   - Worf: Security status and concerns
   - Geordi: Infrastructure and systems status
   - Data: Analytics and insights
   - Troi: User experience and team wellness
   - Crusher: System health and monitoring
   - Wesley: Innovation and prototyping
4. **#1** (1 minute): Synthesis and action items
5. **Captain** (30 seconds): Final direction and approval

## Coordination

- #1 documents action items and owner assignments
- Cross-functional dependencies identified and tracked
- Blockers escalated to Captain for resolution
- Follow-up meetings scheduled as needed

## Best Practices

- Keep updates concise and specific
- Focus on blockers and coordination needs
- Use direct messages for detailed technical discussions
- Document decisions and action items
- Start on time, end on time
