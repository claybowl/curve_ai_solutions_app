# Away Team Mission Command

Deploy a specialized team of 2-3 officers for focused missions requiring cross-domain expertise.

## Usage

```
/away-team [specialist1] + [specialist2] [mission objective]
```

## Examples

**Security Implementation:**
```
/away-team Worf + Geordi Implement secure API gateway with rate limiting and audit logging
```

**Data Infrastructure:**
```
/away-team Data + Geordi Design scalable data pipeline for AI agent telemetry
```

**UX Security:**
```
/away-team Troi + Worf Analyze security user experience and friction points
```

**Product Innovation:**
```
/away-team Wesley + Data Rapid prototype ML-powered feature recommendations
```

## Team Combinations

**Common Away Teams:**
- **Worf + Geordi**: Security architecture and implementation
- **Data + Geordi**: ML infrastructure and data systems
- **Troi + Crusher**: User experience and system health correlation
- **Worf + Wesley**: Security innovation and ethical hacking
- **Data + Troi**: User behavior analytics and insights
- **Geordi + Crusher**: Performance optimization and monitoring

## Protocol

1. Mission briefing specifies team and objective
2. #1 coordinates team deployment and oversight
3. Team members collaborate directly with shared reporting
4. Mission synthesis through #1
5. Captain approval for strategic decisions

## Duration

Typical Away Team missions: 1-3 days for complex implementations, 2-6 hours for analysis tasks.
