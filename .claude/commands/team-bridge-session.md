# Bridge Session Command

Convene all senior officers for collaborative decision-making on major strategic initiatives.

## Usage

```
/bridge-session [strategic topic or decision needed]
```

## Examples

**Strategic Planning:**
```
/bridge-session Q1 strategic planning. I need input on AI agent platform roadmap and resource allocation.
```

**Major Architecture Decision:**
```
/bridge-session Database architecture decision: Multi-tenant vs separate instances per customer. All officers input requested.
```

**Crisis Response:**
```
/bridge-session Major outage post-mortem. Review incident response and recommend system improvements.
```

**Product Direction:**
```
/bridge-session Evaluate entering enterprise market. Assess technical, security, and user experience implications.
```

## Participants

**Always Includes:**
- Captain (Strategic Commander)
- #1 (First Officer - Coordination)
- Data (Analysis & Patterns)
- Geordi (Engineering & Infrastructure)
- Worf (Security & Defense)
- Troi (User Experience & Psychology)
- Crusher (System Health & Monitoring)
- Wesley (Innovation & Prototyping)

## Bridge Session Protocol

1. **Opening**: Captain states strategic objective
2. **Domain Input**: Each officer provides expertise (order: Security → Infrastructure → Data → UX → Health → Innovation)
3. **Synthesis**: #1 synthesizes inputs into options
4. **Decision**: Captain makes final determination
5. **Action Plan**: #1 coordinates implementation with assigned officers

## Duration

Typical Bridge Session: 1-2 hours for discussion, followed by implementation planning.

## Use Cases

- Major architectural decisions
- Strategic planning and roadmap discussions
- Crisis response and recovery planning
- Complex multi-domain problems
- Resource allocation and prioritization
- Product direction pivots
