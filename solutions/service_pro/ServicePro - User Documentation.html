2. Simple Explanation of Key Terms

Let's start by explaining some of the technical terms you'll encounter while using ServicePro. Understanding these terms will help you get the most out of your experience.

Basic Terms Everyone Should Know

AI (Artificial Intelligence)

Think of AI as a smart computer assistant. In ServicePro, the AI learns about your business and helps answer customer questions, book appointments, and provide information without you having to do it manually. It's like having a staff member who works 24/7 and can handle routine customer interactions.

Chatbot

A chatbot is the text-based conversation tool that customers interact with. It's the chat window where customers type questions and receive answers. ServicePro's chatbot uses AI to understand questions and provide helpful, accurate responses just like a human would.

Knowledge Base

The knowledge base is all the information about your business that we teach to the AI. This includes:

  • Your services and their descriptions
  • Pricing information
  • Business policies (cancellation policy, etc.)
  • Frequently asked questions
  • Special offers

Think of it as training material for the AI so it can accurately represent your business to customers.

Dashboard

The dashboard is your control center - where you manage all aspects of your ServicePro system. It's like the admin panel or back office where you can view appointments, customer information, and change settings.

Integration

An integration is when we connect ServicePro to another service or tool you use. For example, we might connect to your Google Calendar so appointments are automatically added there, or to your email service so confirmation emails can be sent automatically.

Technical Terms You'll Encounter

API (Application Programming Interface)

An API is how different computer systems talk to each other. You don't need to understand the technical details, but when we ask for an "API key," we're basically asking for a special password that lets ServicePro securely connect to another service (like OpenAI, weather service, or email service).

API Key

An API key is like a special password or access code that lets one service (ServicePro) connect to another service (like OpenAI or SendGrid). When we ask for an API key, you'll need to create or copy it from the other service's website and paste it into ServicePro.

Webhook

A webhook is an automatic message sent from one service to another. For example, when a new appointment is booked, ServicePro might use a webhook to automatically notify your Google Calendar.

JSON

JSON is a format for storing and exchanging data. You probably won't interact with it directly, but you might see it mentioned in some technical settings or error messages.

Don't worry! You don't need to memorize these terms or understand all the technical details. ServicePro is designed to be user-friendly, and this guide will walk you through each step in plain language.

4. Setting Up Integrations

4.1 OpenAI Setup

OpenAI powers the AI chat functionality and is required for ServicePro to work properly. OpenAI Signup

Getting an OpenAI API Key

  1. Go to https://platform.openai.com/api-keys
  2. Create an account or sign in
  3. Navigate to the API keys section
  4. Click "Create new secret key" Key Types API Key Types:
    OpenAI offers two types of API keys: secret keys and publishable keys (beta). For ServicePro, you need a secret key. Secret keys have full access to your account and should be kept secure.
  5. Give your key a name like "ServicePro Integration"
  6. Copy the API key (it will only be shown once) Important! One-time Display:
    OpenAI only shows the full API key once when you first create it. Be sure to copy it immediately and store it securely. If you lose it, you'll need to create a new key.
curl https://api.openai.com/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Test API key"}]}'
Copied!
Optional: After creating your key, you can test it with the command above (replace YOUR_API_KEY with your actual key).

Setting Up Payment for OpenAI

OpenAI requires payment information even if you're using the free tier:

  1. In your OpenAI account, go to Billing in the left sidebar
  2. Click "Set up paid account"
  3. Add a payment method (credit card)
  4. You can set usage limits to control costs

Entering Your OpenAI API Key

In the Integrations section of the ServicePro setup, paste your OpenAI API key into the designated field.

Cost Information: OpenAI typically provides $5 in free credits for new accounts. After that, you pay for what you use. For most small businesses, this costs approximately $10-30 per month depending on usage.

Important: Your OpenAI API key is sensitive information. Never share it publicly. OpenAI charges based on API usage, so monitor your usage to control costs.

4.2 Google Services Setup

ServicePro can integrate with several Google services including Calendar, Maps, and Sheets. Setting up Google APIs requires creating a service account, which can be confusing, so we've provided detailed instructions below. Google Cloud Console

Creating a Google Cloud Project

  1. Go to Google Cloud Console
  2. Sign in with your Google account
  3. Click on the project dropdown at the top of the page
  4. Click "New Project"
  5. Name your project (e.g., "ServicePro Integration")
  6. Click "Create"
  7. Wait for your project to be created, then select it from the notification or project dropdown

Enabling Required APIs

  1. In your Google Cloud project, go to "APIs & Services" > "Library" from the left menu
  2. Search for and enable these APIs (click each one and press "Enable"):
    • Google Calendar API
    • Google Maps JavaScript API
    • Google Sheets API
    • Google Drive API

Creating Service Account Credentials

  1. Go to "APIs & Services" > "Credentials" from the left menu
  2. Click "Create Credentials" and select "Service Account"
  3. Enter a name for your service account (e.g., "ServicePro Bot")
  4. Add a description like "Service account for ServicePro integration"
  5. Click "Create and Continue"
  6. For Role, select "Project" > "Editor" to grant necessary permissions What is this? Role Permission Explained:
    The "Editor" role gives ServicePro permission to create calendar events, read/write to sheets, and perform other necessary actions. This is required for full functionality. If you prefer more limited access, you can choose specific roles for each service, but setup will be more complex.
  7. Click "Continue" then "Done"
gcloud iam service-accounts create servicepro-bot --display-name="ServicePro Bot"
Copied!
Optional: If you prefer using command line, you can use the command above.

Creating and Downloading a Key

  1. Find your new service account in the list and click on it
  2. Go to the "Keys" tab
  3. Click "Add Key" > "Create new key"
  4. Select "JSON" format and click "Create" Why JSON? JSON Key Format:
    JSON is the recommended format for service account keys as it contains all the necessary credentials in one file. ServicePro's system is designed to work with this format. The P12 format is older and requires additional conversion steps.
  5. A JSON key file will download to your computer - keep this file secure! Security Warning Important Security Note:
    This key file grants access to your Google services. Never share it publicly, commit it to public repositories, or email it unencrypted. If you believe your key has been compromised, return to the Google Cloud Console immediately, delete the key, and create a new one.
gcloud iam service-accounts keys create ~/key.json --iam-account=servicepro-bot@your-project-id.iam.gserviceaccount.com
Copied!
Optional: If you prefer using command line, use the command above (replace "your-project-id" with your actual project ID).

Sharing Calendar/Drive Access

If you're using Google Calendar or Google Drive:

  1. Open your service account details and copy the email address (it looks like: service-account-name@project-id.iam.gserviceaccount.com)
  2. Open Google Calendar
  3. Find the calendar you want to use with ServicePro
  4. Click the three dots next to the calendar name, then "Settings and sharing"
  5. Scroll down to "Share with specific people or groups"
  6. Click "Add people" and paste the service account email
  7. Set permission to "Make changes to events"
  8. Click "Send"

Setting Up API Keys for Maps

  1. In Google Cloud Console, go to "APIs & Services" > "Credentials"
  2. Click "Create Credentials" and select "API Key"
  3. Copy the generated API key
  4. Click "Restrict Key" to limit its usage (recommended)
  5. Under "Application restrictions" select "HTTP referrers" and add your website domain
  6. Under "API restrictions" select "Restrict key" and check "Maps JavaScript API"
  7. Click "Save"

Important: Keep your service account key file secure and never share it publicly. This file grants access to your Google services.

Pricing: Google provides generous free tiers for most services:

  • Google Maps: $200 free monthly credit (about 28,000 map loads)
  • Google Calendar API: No charge for normal usage
  • Google Sheets API: No charge for normal usage

4.3 Email Configuration (SendGrid)

SendGrid enables email communications, including appointment confirmations, reminders, and invoices.

Creating a SendGrid Account

  1. Go to SendGrid Signup Page
  2. Enter your email address, name, and create a password
  3. Complete the signup form with your company information
  4. Verify your email address by clicking the link sent to your inbox

Getting a SendGrid API Key

  1. Log in to your SendGrid account
  2. Go to Settings → API Keys (in the left sidebar)
  3. Click "Create API Key" button
  4. Give your key a name like "ServicePro Integration"
  5. Select "Full Access" or customize permissions (at minimum, you need "Mail Send" permissions)
  6. Click "Create & View"
  7. Copy the API key (it will only be shown once)

Verifying Your Sender Identity

SendGrid requires you to verify your sender identity before sending emails:

  1. In SendGrid, go to Settings → Sender Authentication
  2. Choose either "Single Sender Verification" (easiest) or "Domain Authentication" (more professional)
  3. For Single Sender: Enter your name, email, and company details, then verify via the email you receive
  4. For Domain: Follow the DNS configuration steps provided by SendGrid for your domain

Entering Your SendGrid Information in ServicePro

In the Integrations section of ServicePro setup:

  1. Paste your SendGrid API key into the API Key field
  2. Enter the verified email address you want to use as the sender for customer communications

Free Tier Information: SendGrid offers a free tier that includes 100 emails per day forever, which is sufficient for many small businesses just starting out.

4.4 Payment Processing

ServicePro supports integrated payment processing through Stripe and other payment gateways.

Creating a Stripe Account

  1. Go to Stripe Registration Page
  2. Enter your email address and select a password
  3. Fill in your business information as requested
  4. Verify your email address by clicking the link sent to your inbox

Setting Up Your Stripe Account

  1. Complete the "Activate your account" process with your business and banking details
  2. Add a bank account to receive payments
  3. Set up your branding in the "Branding" section (colors, logo, etc.)

Getting Your Stripe API Keys

  1. In your Stripe Dashboard, go to Developers → API keys
  2. You'll see two types of keys:
    • Publishable key (starts with "pk_") - safe to include in your frontend code Frontend Usage Publishable Key:
      This key is used in your customer-facing payment forms and is safe to expose in your website's code. It allows Stripe.js to create payment tokens but doesn't have permissions to make charges.
    • Secret key (starts with "sk_") - must be kept secure and used only on your server Backend Usage Secret Key:
      This key gives full API access to your Stripe account, including the ability to create charges and issue refunds. It must never be exposed to customers or included in frontend code.
  3. Copy both the Publishable key and Secret key
  4. Enter both keys in the ServicePro integrations settings
VITE_STRIPE_PUBLIC_KEY=pk_test_your_publishable_key
Copied!
STRIPE_SECRET_KEY=sk_test_your_secret_key
Copied!
Format for adding keys to your environment (replace with your actual keys).

Test vs Live Keys: Stripe provides both test and live keys. Test keys (starting with pk_test_ and sk_test_) are for development and don't process real payments. Use Stripe's test cards with these keys. Once you're ready to go live, use the live keys.

Testing Mode: By default, you'll start in Stripe's test mode. This allows you to make test transactions without real money. Toggle between test and live mode using the switch in the dashboard. For testing, use Stripe's test card numbers.

Important: Stripe charges per transaction (typically 2.9% + 30¢ per successful charge). There are no monthly fees or setup costs.

4.5 Twilio SMS Integration

Creating a Twilio Account

  4. Answer Twilio's questions about your use case

Getting Your Twilio Credentials

  2. Enter your Twilio Auth Token
  3. Enter your Twilio Phone Number (with country code, e.g., +12025551234)

Free Trial Information: Twilio provides $15-20 in free credit for new accounts, which is enough for testing and initial setup. After that, messaging costs about $0.0075 per message in the US (prices vary by country).

7. Troubleshooting

Common Issues and Solutions

AI Not Responding

Possible causes: Missing or invalid OpenAI API key, network issues, or API rate limits.

Solution: Check your OpenAI API key in the integrations settings. Ensure you have sufficient credit in your OpenAI account. If using a free tier, you may have reached usage limits.

Appointments Not Syncing

Possible causes: Google Calendar integration issues or incorrect permissions.

Solution: Verify your Google Calendar API credentials and ensure you've granted the necessary permissions. Try disconnecting and reconnecting the integration.

Email Notifications Not Sending

Possible causes: SendGrid API key issue, unverified sender email, or exceeded email limits.

Solution: Check your SendGrid API key and ensure your sender email is verified. If using a free tier, you may have reached sending limits.

Weather Data Not Displaying

Possible causes: Invalid or missing OpenWeather API key, location issues.

Solution: Verify your OpenWeather API key and check that your business location is correctly entered in your business settings.

Getting Additional Help: If you encounter issues not covered in this documentation, please contact support for assistance. Include details about the problem and any error messages you see.
